# Сборка
* Для разработки и тестирования используется docker-compose.yml только с PostgreSQL(запускайте БД отдельно)
* Для продакшена собирается образ приложения (Dockerfile) для дальнейшего деплоя в Kubernetes или на сервер, где уже запущен отдельный PostgreSQL

Для локальной разработки (запускаем только БД, приложение из IDEA):
* docker-compose up -d postgres  # запускаем только PostgreSQL в фоне

Для запуска всего приложения (сборка образа + старт):
* docker-compose up -d  # поднимет и БД, и приложение

Проверка статуса контейнера:
* docker-compose ps (UP значит окей)

Остановит контейнер:
* docker-compose stop

## При запуске flyway.migrate получал ошибку неверного подключения к БД
Причина оказалось в том, что я контейнер для postgres уже запускал ранее, а потому креды уже были созданы.
Чтобы починить, нужно выполнить команды:
1. docker-compose down -v  # Удаляет контейнеры и том с данными
2. docker-compose up -d    # Создает заново с новыми учетными данными
